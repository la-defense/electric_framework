# 更新简报 - 2026-03-29

## 功能更新

### 视觉控制系统完善

#### 1. 视觉数据接管逻辑优化
- **问题**：视觉数据无法控制云台电机移动
- **原因**：视觉接管逻辑被埋在模式判断内部，且电机未自动使能
- **解决方案**：
  - 将视觉接管逻辑提升到最高优先级
  - 当 `vision_recv_data->mode` 为 1 或 2 时，强制设置 `GIMBAL_GYRO_MODE`
  - 自动触发电机使能，无需手动操作

#### 2. 单位转换修复
- **问题**：视觉系统发送弧度制数据，但云台PID控制器需要角度制
- **解决方案**：添加 `RAD_TO_DEG` 宏（57.295779513），自动转换单位
- **影响文件**：`application/cmd/robot_cmd.c`

#### 3. 机械传动补偿
- **问题**：Yaw轴通过1:1.25皮带传动，存在齿轮比
- **解决方案**：
  - 添加 `YAW_GEAR_RATIO 1.25f` 宏定义
  - 视觉yaw角度除以齿轮比后再发送给电机
  - 确保云台转到正确的绝对位置

#### 4. 调试功能增强
- **新增**：在 `DecodeVision()` 函数中添加详细日志
  - 记录接收数据长度
  - 标记帧头位置
  - 报告CRC校验结果
  - 输出解析后的控制数据（mode, yaw, pitch）
- **用途**：快速定位USB通信、数据格式、传输质量问题

## 参数调优

### Pitch轴力矩增强
- **问题**：Pitch轴输出力矩较小，可轻易手动扳动
- **调整**：速度环PID参数 `Kp: 30.0 → 60.0`
- **效果**：显著提升抗干扰能力和响应刚度
- **文件**：`application/config/sentinel_robot/sentinel_config.c`

### Pitch轴角度限幅
- **测量值**：
  - 下限位置：50°
  - 上限位置：-9°
- **配置更新**：
  - `pitch_min_angle: -24.0 → -25.0`
  - `pitch_max_angle: 25.0 → 25.0`
- **作用**：防止机械结构碰撞，保护硬件安全

   - 调整pitch轴速度环PID参数

### 数据流说明
```
视觉系统 → USB → DecodeVision() → recv_data
                                      ↓
                            robot_cmd.c (单位转换 + 齿轮比补偿)
                                      ↓
                            gimbal_cmd_send (角度制)
                                      ↓
                            GimbalTask() (自动使能电机)
                                      ↓
                            DJI电机 (PID控制)
```

## 测试建议

1. **视觉控制测试**
   - 上位机发送 mode=1 或 mode=2
   - 观察云台是否移动到指定角度

2. **力矩测试**

3. **限位测试**

   - 验证电机是否停在限位位置
   - 确认无机械碰撞
